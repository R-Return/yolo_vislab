# 🔬 YOLO VisLab

**An Interactive Bioacoustic Annotation and Research Visualization Suite**

---

## 📖 Overview

**YOLO VisLab** is an advanced, web-based suite tailored for bioacoustic researchers and machine learning practitioners. It provides an intuitive, high-performance interface for visualizing, annotating, and evaluating bounding box predictions (such as those generated by YOLO models) directly on spectrogram images. Complete with synchronized audio playback and integrated filtering capabilities, YOLO VisLab operates entirely client-side. This ensures rapid performance, a seamless user experience, and strict data privacy with zero backend requirements.

## ✨ Key Features

### 📊 Comprehensive Visualization
*   **Bounding Box Inspection:** View ground truth and predicted bounding boxes overlaid on spectrogram images.
*   **Grid & Single-View Layouts:** Toggle between detailed single-image inspection (1x1) and paginated grid layouts (e.g., 3x3, 4x4) for rapid dataset scanning.
*   **Prediction Analysis:** Deep integration for True Positive (TP), False Positive (FP), and False Negative (FN) filtering and analysis, simplifying the evaluation of model performance.

### 🎧 Adaptive Bioacoustic Audio Playback
*   **Click-to-Play:** Click any annotation box (or use the right-click context menu) to instantly play the corresponding audio segment.
*   **Dynamic Frequency Filtering:** Implements native bandpass audio filtering in the browser. The system intelligently matches playback to the highest and lowest frequencies defined by the bounding box geometry.
*   **Freehand Acoustic Probing:** In standard view mode, draw temporary "probe" boxes to instantly listen to specific time-frequency regions without permanently modifying your annotations.

### ✏️ Interactive Annotation & Curation
*   **Precision Editing:** Enter Edit Mode (`E`) for full support of drawing, resizing, and deleting bounding boxes. Newly drawn boxes are automatically selected for immediate adjustment.
*   **Prediction Acceptance:** Quickly convert model predictions into ground-truth annotations during the curation process to accelerate workflow.
*   **Status Tracking:** Clear visual badges ("MODIFIED", "SAVED") track the status of dataset edits in real-time. Edits are securely auto-saved locally using IndexedDB.

### 📈 Evaluation Metrics
*   **F1-Confidence Graph:** Real-time generation of F1-Confidence curves displayed alongside images to help determine optimal operational thresholds for object detection models.
*   **Dataset Statistics:** Instant calculations of precision, recall, and dataset-wide performance.

### 📂 Unified Folder Import
*   **Drag-and-Drop Workflow:** Select a single parent directory containing `audio/`, `images/`, `labels/`, and `predictions/` folders. The suite intelligently scans, processes, and links all corresponding data.
*   **Export as Zip:** Export your curated and modified labels directly to a ZIP file when your annotation session is complete.

## 🚀 Getting Started

### Prerequisites
*   [Node.js](https://nodejs.org/) (v18 or higher recommended)
*   npm or yarn

### Installation & Launch

1. **Clone the repository:**
   ```bash
   git clone <repository_url>
   cd yolo_vislab
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Run the development server:**
   ```bash
   npm run dev
   ```

4. **Open in Browser:** Navigate to `http://localhost:5173` (or the port specified by Vite in your terminal).

## 📁 Expected Dataset Structure

To leverage the unified folder upload feature, organize your dataset within a parent folder as follows:

```text
dataset_folder/
├── images/       # Spectrograms (.png, .jpg)
├── audio/        # Corresponding full audio clips (.wav, .mp3)
├── labels/       # Ground truth annotations (YOLO TXT format)
└── predictions/  # Model output annotations (YOLO TXT format)
```

> **Note:** Filenames across these subdirectories must match exactly (e.g., `recording_001.png`, `recording_001.wav`, `recording_001.txt`) for the suite to link them automatically.

## ⌨️ Keyboard Shortcuts & Controls

A streamlined default control scheme makes curation rapid and effortless:

| Action | Shortcut / Control | Mode |
| :--- | :--- | :--- |
| **Toggle Edit Mode** | `E` | Global |
| **Navigate Pages** | `A` (Previous) / `D` (Next) | Grid View |
| **Select / Focus Item** | Arrow Keys (`↑`, `↓`, `←`, `→`) | Grid View |
| **Enter Focus Mode** | `Enter` | Grid View |
| **Play Box Audio** | Left Click (or Right-click Menu) | Global |
| **Probe Audio Region** | Click & Drag (Draw Temporary Box) | Normal |
| **Select & Adjust Box** | Click & Drag Edges/Corners | Edit |
| **Delete Annotation** | `Backspace` (or Right-click -> Delete) | Edit |
